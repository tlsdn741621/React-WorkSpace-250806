useLocation()
: 리액트 라우터(React Router) 라이브러리의 훅(Hook) 중 하나로,
현재 페이지의 URL에 대한 정보가 담긴 location 객체를 반환합니다.
이 객체를 사용하면
1)현재 경로(pathname),
2)쿼리 스트링(search), 그리고
3) 라우팅 시 전달된 상태(state) 값 등에 접근할 수 있습니다.

주로
1) 현재 URL에 따라 다른 UI를 보여주거나,
2) 페이지 이동 기록을 분석할 때,
또는
3)Link 컴포넌트를 통해 전달된 데이터를 받을 때 유용하게 사용됩니다.

기본 문법
더미 데이터 .json
{
  "pathname": "/contact",
  "search": "?query=react&page=1",
  "hash": "#details",
  "state": { "from": "/" },
  "key": "a1b2c3d4"
}
1) pathname: 현재 URL의 경로를 나타내는 문자열입니다.
(예: /users/123)

2) search: URL의 쿼리 스트링(? 뒤의 부분)을 나타내는 문자열입니다.
(예: ?sort=latest)

3) hash: URL의 해시( # 뒤의 부분)를 나타내는 문자열입니다.
주로 페이지 내 특정 섹션으로 이동할 때 사용됩니다. (예: #section-3)

4) state: Link 컴포넌트나
Maps 함수를 통해 현재 위치로 이동할 때 함께
전달된 데이터를 담는 객체입니다.
페이지를 새로고침하면 사라지는 휘발성 데이터입니다. (예: { from: "/login" })

5) key: 위치(location)마다 부여되는 고유한 키값입니다.
페이지 이동 기록을 관리하는 데 사용됩니다.

=====================================================
useSearchParams()
: URL의 쿼리 스트링(query string)을 읽고
직접 수정할 수 있게 해주는 리액트 라우터 훅입니다.
useLocation과 비슷하지만,
쿼리 스트링을 다루는데 특화되어 있으며
URL을 변경하는 기능까지 제공한다는 점에서 큰 차이가 있습니다.

반환 스타일.
훅을 호출하면 배열(array)을 반환합니다.
이 배열의 첫 번째 요소는 쿼리 파라미터를 읽을 수 있는 객체이고,
두 번째 요소는 쿼리 파라미터를 업데이트하는 함수입니다.
이는 useState와 매우 유사한 방식입니다.

기본 문법
import { useSearchParams } from 'react-router-dom';
const [searchParams, setSearchParams] = useSearchParams();
